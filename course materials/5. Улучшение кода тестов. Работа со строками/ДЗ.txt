Задание №11: Реализовать проверки в fluent-стиле
Во всех тестах для создания, удаления, модификации групп и контактов реализовать проверки в fluent-стиле, используя Hamcrest.

Для этого нужно предварительно сделать контейнер для хранения множества групп или контактов, который добавляет методы withAdded и without (в лекциях показано, как это сделать для групп), а потом использовать этот контейнер в тестах.



Задание №12: Реализовать тест для проверки телефонов, адреса и email-ов
Реализовать тест для проверки телефонов (продемонстрировано в лекциях), почтового адреса и адресов электронной почты на главной странице приложения.

Тест должен проверять только один какой-нибудь контакт, сравнивать информацию о контакте, которая представлена на главной странице, с информацией, представленной в форме редактирования контакта.

Рекомендуется использовать метод обратных проверок.



Задание №13: Реализовать тест для проверки страницы с подробной информацией о контакте
Реализовать тест для проверки страницы с подробной информацией о контакте (она открывается, если в таблице на главной странице кликнуть иконку с изображением человечка в какой-нибудь строке).

Тест должен проверять только один какой-нибудь контакт, сравнивать информацию о контакте, которая представлена на тестируемой странице, с информацией, представленной в форме редактирования контакта.

Рекомендуется использовать метод обратных проверок.

Чтобы упростить задачу, используйте в этом тесте контакт, у которого заполнена только те поля, которые отображаются на главной странице -- имя, фамилия, адреса и телефоны. Остальные поля оставьте пустыми. И не включайте этот контакт ни в какие группы.
